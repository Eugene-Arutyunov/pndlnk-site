# Инструкция: перенос кейса с внешнего сайта в `pndlnk`

## 1) Подготовка

- Найти существующую страницу кейса в `src/projects/<slug>.html`.
- Если страницы нет, создать по шаблону:
  - `{% extends 'layout.html' %}`
  - `{% set pageTitle = '...' %}`
  - `{% block content %} ... {% endblock %}`
- Сразу работать внутри `.ids__wrapper L` и `.ids__text-width`, чтобы не ломать типографику сайта.

## 2) Перенос текста

- Перенести весь основной контент: `h1`, `h2`, абзацы, списки, цитаты.
- Удалить лишнее из исходника (подписки, блоки авторизации, рекомендации, системные элементы сайта-источника).
- Если в цитатах есть служебные пометки вроде `(из интервью)`, убрать их по согласованию.
- Нормализовать кавычки в тексте:
  - писать `ОКБ Понедельник` без кавычек;
  - остальные цитаты/названия в русской типографике оформлять ёлочками `«...»`.
- Критично: кейсы не сокращать и не делать редактуру по смыслу; переносить текст как есть (допустима только техническая очистка мусорных блоков интерфейса источника).

## 3) Подготовка изображений

- Сложить исходные картинки в отдельную папку:
  - `src/assets/projects/<slug>/`
- Если пользователь просит оптимизацию:
  - конвертировать все в `.jpg`;
  - целевой вес файла: ~150–300 КБ;
  - проверить итоговые размеры/вес каждого файла.
- Обложку (если есть отдельная) не вставлять в тело кейса, если это явно оговорено.

## 4) Расстановка изображений в тексте

- Привязывать каждую картинку к абзацу/фрагменту текста, который идет непосредственно перед ней в оригинале.
- Не ориентироваться только на `figcaption`: подписи есть не у всех изображений, поэтому основной якорь — текст перед картинкой.
- Вставлять изображения по смысловым блокам под соответствующими подписями (`figcaption`).
- Если несколько картинок относятся к одной подписи — группировать в одном `figure`.
- Практика для сетки:
  - 1 изображение: 1 колонка;
  - 2 или 3 изображения: одна строка на десктопе, одна колонка на мобилке;
  - 4 изображения: 2x2 на десктопе, одна колонка на мобилке.

### Быстрый чек порядка картинок (3 шага)

1. Взять якорный абзац из оригинала (первые 5-10 слов перед картинкой).
2. Проверить, что после этого абзаца в нашей странице стоит нужный номер файла (`n.jpg`).
3. Открыть страницу в превью и подтвердить визуально: «якорный абзац -> нужная картинка».

## 5) Оформление отзыва (эталон: `/ksc-open/`)

- Структура:
  - `.review`
  - `.review-author` (портрет + имя/роль)
  - `.review-text` (текст отзыва)
- Пока нет фото: использовать серый квадрат-заглушку вместо `img`.
- Имя и роль должны идти перед текстом отзыва.
- Если в исходном кейсе нет отзыва, блок отзыва не добавлять.
- Если отзыв в исходном кейсе есть, переносить и оформлять его в этом паттерне.

## 5.1) Внешние ссылки и сноски

- Сохранять внешние материалы из исходного кейса.
- Оформлять внешние ссылки как короткие сноски на полях справа от основного текста (по аналогии с существующими глоссарийными/полевыми ссылками в проекте).
- Текст ссылки должен быть читаемым названием материала, а URL должен быть встроен в него.

## 6) Финальные элементы страницы

- Добавить навигационную ссылку внизу:
  - `← обратно ко всем проектам` -> `/projects/`
- Перед финальной ссылкой добавить `ids__space L`.
- При необходимости добавить `ids__space L` перед заголовочным блоком (плашка/`h1`) по дизайн-задаче.
- Если у кейса есть отдельная обложка (в нашем процессе это `1.jpg`), поставить её на страницу каталога `src/projects.html` в `project-card__cover` соответствующего кейса (а в тело кейса не вставлять, если так согласовано).

## 7) Проверка качества

- Проверить, что:
  - все картинки открываются по путям `/assets/projects/<slug>/<n>.jpg`;
  - порядок изображений соответствует согласованному списку;
  - нет пустых заглушек `[Изображение будет добавлено]`;
  - нет старой ссылки на источник, если её просили убрать;
  - мобильная сетка не ломается.
- Запустить линтер на измененном файле.

## 8) Что переносить в следующие кейсы

- Использовать текущий кейс как эталон по:
  - структуре текста;
  - группировке изображений в `figure`;
  - оформлению отзыва;
  - нижней навигации обратно к каталогу проектов.
